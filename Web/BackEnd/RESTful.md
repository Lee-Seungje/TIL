# RESTful

REST의 원리를 따르는 시스템을 의미

REST API의 설계 규칙을 올바르게 지킨 시스템을 RESTful하다 말할 수 있다

### 개발 원칙

-   url만으로 내가 제어하고자 하는 것의 종류를 알 수 있어야 한다
-   행위가 명시되어야 한다
-   데이터만 보고 어떤 것인지 알 수 있어야 한다
-   추가적인 정보에 대한 링크를 제공할 수 있어야 한다

### 단점

-   4가지 메소드만 제공한다
-   http에 의존적이다
-   아키텍쳐링 방법만 담고 있다
-   상호작용을 하는 개발에는 적당하지 않다
